## 1. 서비스 개요

### 핵심 컨셉
AI 대화형 인터뷰를 통해 사용자의 막연한 경험을 **구조화된 STAR 블록**으로 변환하는 서비스.

### 핵심 가치
- "빈 칸을 채워라"가 아니라 **"편하게 말해라"**
- 사용자가 모르는 자기 경험의 가치를 **AI가 발견하고 알려준다**
- 한번 만든 블록은 어떤 자소서/면접에든 **재사용 가능한 자산**

### 타겟 사용자
- 취업 준비생 (대학생 3~4학년, 졸업 후 1~2년)
- 이직 준비자 (경력 1~5년)
- 자기소개서/면접 준비에 어려움을 느끼는 사람

## 2. 경쟁 분석 및 차별점

### 기존 도구의 한계

| 기존 방식 | 문제점 |
|-----------|--------|
| ChatGPT에 "자소서 써줘" | 본인 경험이 아닌 AI 소설이 나옴 |
| 자소서 템플릿 사이트 | 빈 칸 앞에서 막힘, 채우기 어려움 |
| 취업 멘토링 | 비용 높고 (회당 5~10만원), 확장 불가 |
| LinkedIn + AI | 회사 매칭은 되지만 경험 구조화는 안 됨 |

### 우리의 차별점
1. **대화형 인터뷰**: 빈 칸이 아닌 자연스러운 대화로 경험 추출
2. **가치 발견**: AI가 사용자도 몰랐던 경험의 강점을 알려줌
3. **구조화된 자산**: 한번 만들면 반복 활용 가능한 경험 블록
4. **역량 커버리지**: 어떤 역량이 부족한지 시각화

## 4. 사용자 플로우

```
[가입/로그인]
     ↓
[메인 대시보드] — 내 경험 블록 목록 + 역량 커버리지
     ↓
[+ 새 블록 만들기]
     ↓
[카테고리 선택] (UI, AI 호출 없음)
  🎓 팀 프로젝트    💼 인턴/직무 경험
  🏆 공모전/대회    🔧 개인 프로젝트
  👥 동아리/대외활동  💪 아르바이트
  📚 자격증/교육    🌏 해외 경험
     ↓
[AI 대화형 인터뷰] (3~5분, 5턴 고정)
  턴 1: 워밍업 — 경험 윤곽 + 기억나는 장면
  턴 2: 상황 파기 — 맥락/규모/제약 조건
  턴 3: 행동 파기 — 구체적 행동 추출 (핵심 턴)
  턴 4: 결과/배움 — 결과 + 배운점 확인
  턴 5: 블록 생성 — 강제 생성 (정보 부족해도 최선의 블록)
     ↓
[블록 확인/수정] — 생성된 블록 검토 후 저장
     ↓
[블록 결과 화면] — 성취감 + 업종/직무 AI 추천 + 역량 현황
     ↓
[자소서 쓰기] or [블록 더 만들기]
     ↓ (자소서 쓰기 선택 시)
[공고 붙여넣기] — 채용 공고 텍스트 통째로 입력
     ↓
[AI 분석] — 회사/문항/JD/글자수 자동 파싱 + 블록↔문항 매칭
     ↓
[자소서 초안 생성] — 문항별 AI 자소서 생성, 인라인 편집, 글자수 표시
     ↓
[전체 복사 / 내보내기]
```

### 순환 구조

              ┌─── 블록 부족한 문항 발견 ───┐
              ↓                           │

              ↑                           │
              └──── 다른 회사 지원 ─────────┘

## 5. AI 대화 설계

### 인터뷰 턴 구조 (5턴 고정)

| 턴 | 목표 | AI 전략 |
|----|------|---------|
| 1. 워밍업 | 경험 윤곽 + 기억나는 장면 | 부담 낮은 질문, 상황+행동 단서 함께 유도 |
| 2. 상황 파기 | 맥락/규모/제약 구체화 | "제일 기억에 남는 순간?" "어떤 상황이었어요?" |
| 3. 행동 파기 | 구체적 행동 추출 (핵심) | "그래서 어떻게 했어요?" + 선택지 제시 |
| 4. 결과/배움 | 결과 + 배운점 보충 | "숫자로 표현하면?" "전후 비교하면?" |
| 5. 블록 생성 | 강제 구조화 | 정보 부족해도 최선의 블록 생성 (예외 없음) |

- 턴 4에서 시스템이 "다음이 마지막 턴" 힌트를 AI에 주입
- 턴 5에서 블록 미생성 시 자동 재요청으로 강제 생성

### 핵심 설계 원칙

1. **질문은 턴당 1~2개만**: 질문 폭탄 금지
2. **공감 먼저, 질문 나중**: "당황스러웠겠네요" → "그래서 어떻게 했어요?"
3. **되받아치기**: 사용자 답변 키워드를 반복하며 관심 표현
4. **"별거 없었어요" 대응**: "이건 좀 아닌데 싶었던 적 있어요?" 로 리다이렉트
5. **가치 리프레이밍**: "3명이서 B+이면 잘한 거죠"

### 분기 처리

```
사용자가 짧게 답변할 때:
  → "조금만 더 구체적으로 말해주시면 좋은 블록이 될 것 같아요"
  → 선택지 제시: "혹시 이런 상황이었나요? A) ... B) ..."

사용자가 여러 경험을 섞을 때:
  → "두 가지 다 좋은 소재인데, 하나씩 정리하면 어때요? 어떤 걸 먼저 해볼까요?"

사용자가 감정적일 때 (부정적 경험):
  → "힘든 경험이었지만, 그 속에서 배운 게 있다면 그게 가장 강력한 소재가 됩니다"
```

## 7. DB 스키마 (핵심)

### experience_blocks
```sql
- id (uuid, PK)
- user_id (uuid, FK → auth.users)
- category (text) -- 팀프로젝트, 인턴, 공모전 등
- title (text) -- 블록 제목
- situation (text) -- 상황
- action (text) -- 행동
- result (text) -- 결과
- lesson (text) -- 배운점
- tags (text[]) -- #위기대응, #리더십 등
- recommended_questions (text[]) -- 추천 문항 유형
- strength_score (int) -- 경험 강도 1~5
- ai_insight (text) -- AI가 발견한 가치 포인트
- created_at, updated_at
```

### interview_sessions
```sql
- id (uuid, PK)
- user_id (uuid, FK → auth.users)
- block_id (uuid, FK → experience_blocks, nullable)
- category (text)
- messages (jsonb) -- [{role, content, timestamp}]
- status (text) -- in_progress, completed, abandoned
- turn_count (int)
- created_at, completed_at
```
